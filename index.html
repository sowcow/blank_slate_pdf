RUE.pdf



      TODO: ordering + clearing, also exact coordinates use example(s)

      PDF idea with ternary plot that supports done/todo-lists of ~10 items where every item falls into one of three categories.

      The idea is one can group done items of the day into three categories, for example Recovery, Utilization, and Expansion (of any capacity/asset), and then having categories plotted for summary at the end of the month.
      Use ternary plot page by writing days as numbers at right coordinates, producing summary for later review.

      Circle page is a short-term scratchpad, use it by opening the main circle page, it has invisible links inside the circle, then enter random link at any point within the circle, going next/previous from within scratchpad page is similar to to going back and entering another random point because links positions within the circle have no relation to pages order.
      "-" sequence of pages can be used for title page, any more high-level content on the whole scope of the month, or catchall for later use.
      "+" pages are for days of month one page per day, implied is that they will have todo/done items of three categories, so count of done items will go into the summary plot by using numbers as coordinates.

      One consequence of using ternary plots is that there should be multiple ternary plots because sum of coordinate values is a fixed number and every ternary plot supports only one such number.
      Therefore there are multiple plots, some days will go into different depending on the overall count of items done.
      I have only some plots to make the most use of space within three consecutive pages I allocated for them.
      In case when number of items done is above 10 then it's possible to draw them instead of points as areas contained by lines of coordinates, this needs testing for usability, for example 3*9 = 27 items would be earea covering almost whole triangle, just except corners since it will go by lines marked with 9 there.
      In case number of items is odd when available plots are only even then one should creatively resolve this somehow.

      Actually because there is a sequence of plots overall, one could say it records 4 dimensions, even though they are not independent from each other.
      So RUE.pdf must be the first 4D PDF.

Teeth.pdf



      Teeth.pdf is inspired by After Hours and has a room for The Shining too.
      Main "+" view is a monthly grid with further daily todo lists
      (can be Teeth or Sandworm version or can have both depending on selected options).
      The "#" view is a maze - place for catchall pages for unrelated ideas,
      they can be stored in further Δ pages.
      The "-" view starts from the first page of the PDF,
      it is for higher level content like conception of usage of the PDF,
      or possibly for review-type notes accumulation, it's the only multi-page entry.

      Below are the settings for Make Teeth;
      there is an option to have cryptic review pages (Space Odyssey inspired);

      Most recent and new default option is to use Sandworm todo list:
      If "+" page is used like a monthly calendar then you have one Sandworm per day.
      Inside of the Sandworm you have very limited space for todo list, day focus or similar content.
      Outside of it there is clockface with 12 squares to do time planning or accounting.
      Whether to slay the Sandworm at the end of the day or to feed maidens into it is the discretion of the user.

      I have gray colored lines by default for Remarkable2,
      use any hex color code if RM-PRO is the target device,
      like orange is used for other PDFs below.

      If multiple options are selected then they are accessible by turning the page.
